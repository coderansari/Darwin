# Darwin — Dashboard

A premium, mobile-first dashboard for the Darwin evolutionary trading-strategy
foundry. Every number is the **real** locked-champion backtest, read from
`public/data/darwin.json` (regenerated by `scripts/export_dashboard.py` in the
repo root). No mock data.

Built with Next.js 14 (App Router) + Tailwind + Recharts + Framer Motion.
Statically generated — instant load, perfect for a live demo.

## Develop

```bash
cd web
npm install
npm run dev          # http://localhost:3000
```

## Refresh the data

From the repo root, re-run the champion backtest and rewrite the data file:

```bash
python -m scripts.export_dashboard   # writes web/public/data/darwin.json
```

Then rebuild (`npm run build`) — the JSON is baked into the static page.

## Live features (optional)

The dashboard is fully static by default. Two optional **server-side** routes add
live touches — both read keys server-side only (never shipped to the browser):

- **`POST /api/evolve`** — the "Evolve live" button. Runs Claude as the GA's real
  mutate operator on the locked champion and returns a fresh, validated strategy spec.
  Honors `ANTHROPIC_API_KEY`, optional `ANTHROPIC_BASE_URL` (e.g. the DGRID gateway),
  and `DARWIN_MODEL` (defaults to `claude-opus-4-8`).
- **`GET /api/sentiment`** — the live Fear & Greed badge. Server-side CMC fetch via
  `CMC_PRO_API_KEY`.

Configure locally in `web/.env.local` (git-ignored; see `.env.example`). If keys are
absent, the routes degrade gracefully and the dashboard stays fully static.

> ⚠️ Never set `NEXT_PUBLIC_` on these — that exposes the key. Never deploy
> `BSC_PRIVATE_KEY` or TWAK secrets to the web app.

## Deploy to Vercel

1. Push the repo to GitHub.
2. In Vercel → New Project → import the repo.
3. Set **Root Directory = `web`** (everything else auto-detects: framework
   Next.js, build `next build`).
4. Deploy. The link preview (OpenGraph card) is generated in code at
   `app/opengraph-image.tsx` — no asset upload needed.

## Branding assets (swap-in)

The header/footer use a built-in placeholder DNA-helix mark
(`components/brand/Logo.tsx`) and `public/icon.svg`. To use the final generated
logo, replace `public/icon.svg` (favicon) and, if desired, swap the SVG paths in
`components/brand/Logo.tsx`. For a custom social card, replace
`app/opengraph-image.tsx` or drop a static `public/og.png` and re-reference it in
`app/layout.tsx`.

