# Darwin — Demo Walkthrough

A 2–3 minute demo that shows the whole pipeline. Run these live, or record them.

## 0. Setup (once)

```bash
pip install -r requirements.txt
cp .env.example .env          # fill in keys for the LIVE demo (optional)
```

Darwin runs **with or without keys** — no keys = deterministic synthetic data so
the demo never fails on camera. With keys it uses live CoinMarketCap data + Claude.

## 1. One-command demo

```bash
python -m scripts.demo
```

Narrates four stages: **Data → Evolve → Champion → Execute**. Point out:
- the **CMC Fear & Greed** reading driving sentiment-aware strategies,
- fitness **climbing every generation** (evolution working),
- the champion's **Sharpe / max-drawdown / 100% rule-adherence**,
- the **rule-gated execution plan** (allowlist = the spec's universe).

## 2. Evolve a champion (the headline)

```bash
python -m darwin.cli evolve --universe BTC,ETH,BNB,SOL --generations 8
```

Then open the generated `runs/<id>/report.md` — metrics table, equity-curve
sparkline, per-generation fitness progression, and the hall of fame.

## 3. Show the MACD + Fear & Greed strategy spec

```bash
python -m darwin.cli backtest --spec strategies/sample-champion.json
```

Open `strategies/sample-champion.json` to show the readable, backtestable spec —
this is the **Track 2 deliverable**.

## 4. The sponsor integrations

```bash
# Trust Wallet Agent Kit — live swap route for the champion's tokens (needs TWAK creds):
python -m darwin.cli execute --spec strategies/sample-champion.json

# BNB AI Agent SDK — register the agent's ERC-8004 identity on BSC testnet (needs BSC key):
python -m darwin.cli register --name darwin-foundry
```

For the on-chain proof, capture the **transaction hash** printed by `register`
and show it on `testnet.bscscan.com`.

## What to say

> "Everyone prompts an LLM once for a strategy. Darwin breeds a *population* of
> them — generated by Claude from CoinMarketCap data, scored on a deterministic
> backtester against the exact judging rubric (risk-adjusted return, drawdown,
> rule adherence), and evolved over generations. The champion is a readable,
> backtestable spec that also executes on BSC through Trust Wallet and registers
> its own on-chain identity via the BNB AI Agent SDK."
